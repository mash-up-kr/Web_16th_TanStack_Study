# 🐦‍🔥 웹 16기 TanStack 스터디 🔥

## 📝 스터디 규칙

### 진행 방식 <br/>

- 공통 주제 + @: 매주 정해진 공통 주제를 학습하되, 개인별로 해당 도메인의 서브 주제를 선정하여 깊이 있게 탐구하는 것을 목표로 합니다.

- 스터디 당일 룰렛을 통해 선정된 2명이 발표를 진행합니다.
- 주차별로 스터디 전/후로 질문 혹은 코멘트를 작성합니다. 해당 질문들은 스터디 도중 및 스터디 종료후에 답변합니다.

- 파일은 기본적으로 `WHAT-HOW-WHY`구조로 작성합니다.
  - WHAT: 이 기능/패턴은 무엇인가?

  - HOW: 어떻게 사용하는가?

  - WHY: 왜 이 방식인가?

  ```markdown
  ## 1. 이게 무엇인지

  ---

  ## 2. 어떻게 사용하는지

  ---

  ## 3. 왜 써야하는지

  ---
  ```

- 주차별 파일 업로드 구조 예시는 아래와 같습니다.
  ```
  1주차-TanStack 철학
  ├── 최경일
  ├───── README.md
  ├── 정혜인
  ├───── README.md
  ├── 이용훈
  ├───── README.md
  ├── 한유진
  └───── README.md
  ```
  → 해당 이슈에 파생하는 개인별 PR 생성 -
  질문은 PR에 달기

### 📚 스터디 방식

1. 각자 자신의 **GitHub 아이디 브랜치**를 생성합니다.
2. 해당 브랜치에서 '1주차/이름/README.md' 형태로 폴더를 생성하여 정리한 내용을 올립니다.
3. `main` 브랜치로 **Pull Request**를 생성합니다. (PR 제목: `docs: [N주차] 이름 - 학습 주제`)
4. 스터디 완료 후 다른 사람의 PR에 코멘트를 작성합니다.
5. PR은 마지막에 확인(approve)한 사람이 머지(merge)를 진행해주세요.

## 주차별 주제

- `1주차` - TanStack의 설계 철학 & 탄생 배경
  - Query/Router/Table/Virtual 각각 왜 만들었는지, 기존 솔루션의 어떤 불편함을 해결했는지
  - Headless / Framework-agnostic 전략을 왜 택했는지
- `2주차` - Query
  - Observer 패턴 기반 상태 구독 구조 (QueryObserver 내부 흐름)
  - staleTime / gcTime / invalidation 캐시 생애주기
- `3주차` - Router
  - 타입-세이프 라우팅 구조 (params/search 타입 추론 원리)
  - Loader와 데이터 프리패칭 전략, Query와의 통합 패턴
- `4주차` - Table
  - Headless 구조와 렌더링 최적화 원리
  - 상태 관리 분리, 리렌더링 최소화 방식
- `5주차` - Virtual
  - 가상화 핵심 원리 (overscan, measureElement, 동적 높이 처리)
  - Table과의 결합 패턴, 행/열 가상화 내부 계산 방식
- `6주차` - Intent
- `7주차` - Start
